SOFTWARE REQUIREMENTS SPECIFICATION (SRS)
Project: SERVEASE – Service Booking Platform
1. INTRODUCTION
1.1 Purpose

The purpose of this document is to provide a detailed description of the Servease system. It includes functional and non-functional requirements, system behaviour, constraints, and operational details. This SRS is intended for developers, evaluators, and stakeholders to understand the system completely.

1.2 Scope

Servease is a desktop-based service booking platform that connects users with service providers. It allows users to browse services, book them, and provide feedback, while providers can manage services and bookings. An admin manages users, providers, services, and overall system operations.

1.3 Definitions
User: Customer who books services
Provider: Person offering services
Admin: System controller managing platform
Booking: Reservation of a service
Service: Offering provided by providers
Review: Feedback given by users
2. OVERALL DESCRIPTION
2.1 Product Perspective

Servease is a standalone Java Swing application connected to a relational database. It follows MVC architecture:

UI Layer (Swing)
Controller Layer
Service Layer (Validation)
DAO Layer (Database access)
